# Dotfiles

このリポジトリは、macOS 環境における開発環境（zsh, VS Code, Antigravity, Karabiner-Elements）の設定ファイルを一元管理・自動配置するためのオートメーションツールです。

## ✨ 特徴 (Features)

- **セットアップの自動化**: `install.sh` を実行するだけで、各アプリの適切なディレクトリへシンボリックリンクを配置します。
- **安全なバックアップ機能**: 既存設定を破壊しないよう、実ファイルが存在する場合は自動で `.bak` 拡張子を付けて退避させます。
- **Fail-fast 設計**: 実行中のエラー（権限不足、パスミス等）を即座に検知し、適切なリカバリ案内を表示して停止します。
- **マルチ IDE 同期**: VS Code の設定資産を Google Antigravity でも参照できるよう、共通化・同期させています。

## 📦 収録内容 (Contents)

- **`.zshrc`**: ターミナル環境のパーソナライズ設定
    - 補完機能の強化（大文字小文字の区別なし、Tabキーによる選択UI）
    - Git ステータスの動的表示（カレントブランチの可視化）
    - 安全策の導入（`rm`, `cp`, `mv` 実行時の確認プロンプト強制）
    - 開発ツール（Homebrew, rbenv）の初期化とパス設定
- **`vscode/`**: VS Code の共通設定資産
    - `settings.json`, `keybindings.json` を一元管理
- **`karabiner/`**: Karabiner-Elements の設定
    - Ctrl + F/B/P/N を物理矢印キーに変換（Emacs風ナビゲーション）
    - Caps Lock を Left Control へ変換
- **`install.sh`**: 環境構築用オートメーションスクリプト
    - シンボリックリンク配置、既存設定の自動バックアップ、エラーハンドリングを実装
- **`.gitignore`**: リポジトリのクリーン化設定
    - macOS 特有の一時ファイル（`.DS_Store` 等）の混入を防止
## 🛠 使い方 (Usage)

### 1. リポジトリのクローン
ターミナルを起動し、ホームディレクトリ（`~/`）など、管理しやすい場所にクローンします（`~/dotfiles` として配置するのが一般的でおすすめです）。

```zsh
# ホームディレクトリへ移動
cd ~/

# リポジトリのクローン
git clone https://github.com/あなたのユーザー名/dotfiles.git

# ディレクトリへ移動
cd dotfiles
```
### 2. インストールスクリプトの実行
実行権限を付与してからスクリプトを起動します。
```zsh
# 実行権限の付与 (chmod: change mode)
chmod +x install.sh

# スクリプトの実行
./install.sh
```

### 3. 設定の反映 (Apply Changes)
VS Code の設定は自動的に反映されますが、zsh の設定を有効にするには、ターミナルで以下のコマンドを実行するか、ターミナルを再起動してください。

```zsh
source ~/.zshrc
```

## 📐 設計思想と AI との共創 (Design Philosophy & AI Collaboration)

本プロジェクトは、生成AI（Gemini）を「ペアプログラマー」として活用し、人間の設計指針に基づき開発されました。単なるコード生成に留まらず、以下の技術的原則を徹底することで、実務レベルの品質を追求しています。

### 技術的ポイント
- **DRY (Don't Repeat Yourself)**: リンク作成とバックアップのロジックを共通関数化し、メンテナンス性を高めています。
- **冪等性 (Idempotency)**: 何度実行しても常に「あるべき状態」に収束するよう設計されており、環境の更新時も安全に再実行可能です。
- **UXを意識したエラーハンドリング**: `trap` コマンドによるエラー監視を実装。異常終了時に「何を確認すべきか」をユーザーに明示する、親切な CLI インターフェースを目指しました。

### 人間による品質担保
AI の出力をそのまま受け入れるのではなく、人間が以下のプロセスを主導しています。
- **要件定義**: macOS 固有のパス解決や、複数 IDE の設定同期といった具体的な課題を定義。
- **コードレビュー**: 生成されたコードの安全性（`ln -sf` の挙動など）を確認し、実環境でテストを実施。
- **ドキュメンテーション**: 「How（何をしているか）」だけでなく、初心者にも伝わる「Why（なぜそうしたか）」をコメントに付与し、ナレッジの共有性を高めています。
